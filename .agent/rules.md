# Regras de Desenvolvimento do Projeto

- **MODELO GEMINI**: Nunca mude o modelo do Gemini especificado no código. O modelo atual e obrigatório é o `gemini-2.5-flash`.
- **PADRÕES DE COMMIT**: Use sempre **Conventional Commits** (ex: `feat:`, `fix:`, `docs:`, `chore:`) e escreva as mensagens obrigatoriamente em **Português**.
- **QUALIDADE DE CÓDIGO**:
    - Siga rigorosamente os princípios de **Clean Code**.
    - Comente o código de forma clara e útil.
    - Documente funções e classes usando Docstrings seguindo as melhores práticas de engenharia de software (ex: padrão Google ou NumPy).
- **IDIOMA**: Toda a interface de saída (logs, outputs estruturados) e documentação interna devem ser em **Português do Brasil**.
